# Roteiro de Vídeo Pitch - Personalização Web

**Tempo Estimado:** 3:30 a 4:00 minutos
**Cenário:** Barbearia Vintage Local
**Objetivo:** Demonstrar a solução técnica criada para o trabalho acadêmico.

---

## 1. Abertura e Apresentação do Problema (0:00 - 0:45)

**[Visual]**: Câmera focada em você (host). Fundo neutro ou escritório.
**[Ação]**: Fale com entusiasmo e clareza.

> "Olá, professores e colegas! Meu nome é [Seu Nome] e hoje vou apresentar o projeto prático sobre Personalização e Integração Web."

> "O cenário que escolhi foi o da 'Barbearia Vintage', um pequeno negócio local que, como muitos outros, sofre com o orçamento apertado para tecnologia. Eles precisavam urgentemente sair do agendamento por telefone e WhatsApp manual, mas não tinham verba para pagar um desenvolvedor sênior para criar um app do zero."

> "O desafio era: Como entregar uma solução profissional, funcional e bonita, gastando pouco?"

---

## 2. A Solução e Demonstração (0:45 - 2:00)

**[Visual]**: Screencast (gravação de tela). Mostre a ferramenta No-Code (Webflow ou Softr) e depois o resultado final no navegador.

> "A resposta para isso foi unir o melhor de dois mundos: a agilidade das ferramentas No-Code para o visual do site, com a inteligência do 'Low-Code' para a funcionalidade específica."

**[Ação no Vídeo]**: Navegue pela página. Abra o widget de agendamento. Simule o preenchimento errado para mostrar o erro.

> "Aqui vocês podem ver o site da barbearia. Mas o coração do projeto é este Widget de Agendamento Personalizado que desenvolvi. Diferente dos formulários padrões da ferramenta, este aqui possui inteligência."

> "Vejam só: se eu tento agendar sem colocar o nome, ou com uma data no passado... [Mostre o erro vermelho na tela]. O sistema me bloqueia instantaneamente. Isso é JavaScript puro rodando no navegador do cliente, garantindo que os dados cheguem limpos para a barbearia."

---

## 3. Destaques Técnicos (2:00 - 3:15)

**[Visual]**: Screencast mostrando o código no VS Code (ou editor de texto). Dê zoom nas partes importantes.

> "Agora, vamos olhar 'debaixo do capô'. Para criar essa experiência, utilizei os três pilares da Web:"

> "Primeiro, **HTML Semântico**. Não usei apenas 'divs'. Usei tags de formulário reais, o que ajuda na acessibilidade."

> "Segundo, **CSS Responsivo**. [Mostre a tela do navegador diminuindo para tamanho de celular]. Percebam como o formulário se adapta. Usei variáveis CSS e unidades relativas para garantir que o cliente consiga agendar pelo celular sem dificuldade."

> "E terceiro, o **JavaScript**. Criei uma simulação de API usando Promises. Quando clico em 'Agendar', o botão muda de estado, mostra um 'loading'... isso dá feedback visual ao usuário, algo crucial para uma boa experiência."

---

## 4. Dificuldades e Conclusão (3:15 - 4:00)

**[Visual]**: Volta para a câmera em você.

> "A maior dificuldade desse projeto foi garantir que o meu código não quebrasse o layout da ferramenta No-Code. Tive que isolar bem os estilos CSS para evitar conflitos."

> "Concluo que o conhecimento em padrões web é fundamental, mesmo na era do No-Code. É esse conhecimento que nos permite pegar uma ferramenta simples e transformá-la em uma solução profissional e sob medida."

> "Obrigado pela atenção!"
